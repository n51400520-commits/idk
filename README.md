# Практика: AS-IS және TO-BE модельдері

## 1. AS-IS модель («Қалай бар»)

```mermaid
graph TD
  A[Клиент сайтта өтінім қалдырады] --> B[Менеджер деректерді Excel-ге көшіреді]
  B --> C[Қоймаға қоңырау шалады]
  C --> D[Шотты Word-та толтырады]
  D --> E[Бухгалтер төлемді қолмен тексереді]

```
## 2. TO-BE модель («Қалай болады»)
```mermaid
graph TD
  A[Клиент тапсырыс береді] --> B[CRM жүйесіне автоматты түрде түседі]
  B --> C[Жүйе қоймадағы қалдықтарды тексереді]
  C --> D[Шот автоматты түрде қалыптасады]
  D --> E[Төлем мәртебесі банкпен интеграция арқылы жаңарады]

```
## 3. Экономикалық негіздеме

| Көрсеткіш | Мәні |
|----------|------|
| CAPEX (инвестиция) | 2 000 000 ₸ |
| Ай сайынғы үнем | 400 000 ₸ |
| PBP (өзің-өзі ақтау мерзімі) | 5 ай |
| ROI (жылдық қайтарым) | 140% |
